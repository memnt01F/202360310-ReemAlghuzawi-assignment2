# Assignment 2 – Interactive Portfolio Website

## Project Overview

This project is an enhanced version of my personal portfolio website developed in Assignment 1.  
The goal of Assignment 2 was to improve the website by adding interactivity, dynamic behavior, animations, and user feedback using JavaScript.

The website demonstrates modern web development practices, including dynamic content updates, form validation, responsive design, and smooth user interactions.

---

## Features Implemented

### 1. Dynamic Content – Project Search Filter

A search feature allows users to filter projects in real time while typing.

Example:
- Users can type keywords into the search bar
- Matching projects remain visible
- Non-matching projects are hidden
- A message appears if no projects are found

This improves usability and makes the website more interactive.

---

### 2. Data Handling – Form Validation

The contact form includes input validation using JavaScript.

Behavior:
- Prevents submission if fields are empty
- Displays an error message for missing input
- Shows a success message when the form is submitted correctly
- Resets the form after submission

---

### 3. Animation and Transitions

The website includes smooth visual effects to improve the user experience.

Examples:
- Hover effects on project cards
- Fade-in animation for project elements
- Smooth transitions when interacting with content

These animations make the interface more modern and engaging.

---

### 4. User Feedback

The website communicates clearly with users through messages.

Examples:
- "Searching..." message while filtering projects
- "No projects found" message when search results are empty
- Error message for invalid form input
- Success message after form submission

---

## Technologies Used

- HTML5 – Structure and content
- CSS3 – Styling, layout, and animations
- JavaScript – Interactivity and dynamic behavior
- CSS Grid – Project layout
- Flexbox – Navigation layout
- LocalStorage – Theme preference persistence
